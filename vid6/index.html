//CLS-Cumulative Layout Shift // It is a measure of how much the content of a webpage shifts around as it loads. A high CLS score indicates that elements on the page are moving around, which can be frustrating for users and lead to a poor user experience. To improve CLS, web developers can optimize their code and ensure that elements are properly sized and positioned before they are loaded. This can help to reduce the amount of layout shift and improve the overall user experience on the website. //LCP-Largest Contentful Paint // It is a metric that measures the time it takes for the largest content element on a webpage to become visible to the user. This metric is important because it gives an indication of how quickly the main content of a webpage is loading, which can impact the user experience. A fast LCP can lead to a better user experience and higher engagement, while a slow LCP can lead to frustration and a higher bounce rate. To improve LCP, web developers can optimize their code and ensure that the largest content element is properly sized and positioned for fast loading. //FID-First Input Delay // It is a metric that measures the time it takes for a webpage to respond to the first user interaction, such as clicking a button or entering text into a form. A low FID indicates that the webpage is responsive and provides a good user experience, while a high FID can lead to frustration and a poor user experience. To improve FID, web developers can optimize their code and ensure that the webpage is responsive and can quickly respond to user interactions.
